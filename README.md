Trade-Offs: (Sacrificios y Beneficios) 

4.1 Escalabilidad VS. Complejidad
•	Beneficio: Escalabilidad horizontal al añadir más instancias de servicios según demanda.
•	Sacrificio: Incremento en la complejidad inicial debido a la configuración de múltiples servicios.

4.2 Modularidad VS. Tiempo 
•	Beneficio: Mantenimiento más sencillo y capacidad de extender funcionalidades.
•	Sacrificio: Desarrollo más lento en la fase inicial por la necesidad de dividir sistemas en módulos independientes para un trabajo un poco rápido.
4.3 Consistencia VS. Disponibilidad
•	Beneficio: Alta disponibilidad asegurada incluso durante picos de carga.
•	Sacrificio: Uso de una consistencia eventual en algunos módulos para intentar mejorar el rendimiento o solucionar problemas de conexión.
